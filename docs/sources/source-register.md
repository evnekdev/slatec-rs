# Authoritative source register

This register identifies the primary materials used to document the SLATEC Common Mathematical Library (CML) and major packages represented in it. Stable identifiers correspond to `metadata/sources.toml`. Retrieval dates record when the resource was opened or otherwise verified, not its publication date.

## Authority tiers

- **Tier 1 — canonical distribution or original project publication:** the Netlib library itself, original project manuals, source files, or records issued by the responsible organization or authors.
- **Tier 2 — authoritative institutional context:** official institutional catalogues, standards, or preservation copies that describe or host the original material.
- **Tier 3 — secondary orientation:** reputable historical or explanatory sources. Tier 3 sources may guide discovery but must not override Tier 1 evidence.

## Release and change evidence

No single authoritative, library-wide changelog covering all SLATEC releases was verified. For now, release/change research must use the following evidence hierarchy:

1. release identity in `netlib-slatec-index` and `netlib-slatec-readme`;
2. dated revision-history sections in individual source prologues from `slatec-source-tree`;
3. package-specific source histories and guide editions;
4. archive inventories and checksums;
5. secondary historical accounts only as clearly labelled supplements.

A future ingestion pass should search the complete archive for version banners, `REVISION HISTORY` blocks, and package acknowledgements, then generate a provenance-linked change dataset rather than inventing a unified chronology.
